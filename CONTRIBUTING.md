# Contributing to ScamBuster

Thank you for your interest in contributing to ScamBuster!

## Current Contribution Model

ScamBuster is currently in **private development** as part of a Master project. The operational code is not publicly available for security reasons.

However, there are several ways you can contribute:

---

## Ways to Contribute

### 1. Documentation Improvements

This preview repository is public. You can contribute by:

- **Reporting issues**: Typos, unclear explanations, broken links
- **Suggesting improvements**: Better diagrams, additional FAQ entries
- **Translations**: Help make documentation accessible in other languages

To contribute:
1. Open an issue describing the improvement
2. If approved, submit a pull request with your changes

### 2. Research Collaboration

We welcome collaboration on:

- **Methodology validation**: Independent evaluation of our approach
- **Dataset analysis**: Once released (February 2026)
- **Comparative studies**: Benchmarking against other systems
- **Extension research**: Building on our work

To collaborate:
1. Contact us with your research proposal
2. We'll discuss scope and data sharing
3. Formal collaboration agreement if needed

### 3. Pilot Programs

Organizations can participate in pilot programs to:

- Evaluate ScamBuster in their environment
- Provide feedback on usability and effectiveness
- Influence roadmap priorities
- Early access to new features

To participate:
1. [Contact us](README.md#contact) with your use case
2. We'll discuss scope and success criteria
3. Sign pilot agreement (includes NDA)

### 4. Open Source Components (Coming Q1 2026)

We plan to open-source non-sensitive components:

- **IOC normalization utilities**: Parsing and standardizing indicators
- **STIX mapping helpers**: Converting IOCs to STIX 2.1 format
- **Evaluation scripts**: Statistical analysis tools

Once released, these will accept contributions via standard GitHub workflow.

## Code of Conduct

All contributors must adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).

Key principles:
- **Respect**: Treat all participants with respect
- **Collaboration**: Work together constructively
- **Responsibility**: Consider the impact of your contributions
- **Security**: Never compromise operational security

---

## What We're NOT Looking For

To protect operational security, we cannot accept:

- Contributions that reveal operational details
- Scambaiting prompts or tactics
- Real scammer data or identifiers
- Automation workflows or playbooks

These restrictions are necessary to prevent the system from being evaded by adversaries.

---

Contributors will be recognized in:

If you're unsure how to contribute or have ideas not covered here:
